# React — Writing Markup with JSX (ep03)

JSX is a syntax extension that lets you write HTML-like markup inside JavaScript.

React components use JSX to describe what to render on the screen.

JSX looks like HTML, but it has some important rules to follow.

First rule: return a single root element. If you have multiple elements, wrap them in a parent tag or use a fragment with angle brackets.

Second rule: close all the tags. Self-closing tags like image need a forward slash. List items and other tags must have closing tags.

Third rule: use camelCase for attribute names. The class attribute becomes className. Stroke-width becomes strokeWidth.

This is how React converts HTML into JSX. It looks similar, but these rules prevent errors in your code.

JSX makes React components clearer because markup and logic live together.

Converting HTML to JSX is easy once you know the rules.
