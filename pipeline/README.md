# Run the Docs — Production Pipeline

**One integrated system. TTS duration is the single source of truth.**

## The 4-Step Flow

```
Step 1: TTS               Step 2: Build HTML        Step 3: Render Frames     Step 4: Mix A/V
┌─────────────────┐      ┌──────────────────────┐   ┌────────────────────┐   ┌──────────────┐
│ tts.py          │      │ build.py             │   │ render.py          │   │ ffmpeg mix   │
│ - Kokoro voices │──┬──→│ - Read timing.json   │──→│ - Read timing.json │──→│ - Audio wav  │
│ - Outputs:      │  │   │ - Embed TOTAL_DURATION   │ - Render frames    │   │ - Video mp4  │
│   voice.wav     │  │   │ - Write animation.html   │ - Output: frames/  │   │ - Output mp4 │
│   timing.json   │  │   └──────────────────────┘   └────────────────────┘   └──────────────┘
└─────────────────┘  │
     ↓ (60s audio)   │
  total_duration     └── drives TOTAL_DURATION value
  total_frames_30fps └── drives frame count in render.py
```

## Critical Concept: One Source of Truth

**TTS output → timing.json contains:**
- `total_voice_duration`: How long the narration is (e.g., 60.976s)
- `total_duration`: Voice + outro buffer (e.g., 64.976s = 60.976 + 4)
- `total_frames_30fps`: Exact frame count (e.g., 1950 = 64.976 × 30)
- `segments`: Array of `{start, end, duration, text}` for each sentence

**This single JSON file drives:**
1. **build.py** → writes `TOTAL_DURATION` into the HTML animation code
2. **render.py** → reads `total_frames_30fps` to know how many frames to render
3. **ffmpeg** → combines video + audio (both same length, no gaps)

**Never hardcode durations. Never guess frame counts. Read from timing.json.**

## Step-by-Step: Making an Episode

### 1. Write the Script

Create `series/<series>/production/ep<NN>/script.md`:
```markdown
# Episode <NN>: Title

Intro silence: 2.0s

## Sentence 1
"Your narration here."

## Sentence 2
"More narration."

...

Outro: 4.0s silence
```

### 2. Copy Pipeline Templates

```bash
cp pipeline/template_tts.py series/react/production/ep02/tts.py
cp pipeline/template_build.py series/react/production/ep02/build.py
cp pipeline/template_render.py series/react/production/ep02/render.py
cp pipeline/build_episode.sh series/react/production/ep02/build_episode.sh
```

Edit each script to match your episode narration and animation design.

### 3. Run the Pipeline

```bash
cd series/react/production/ep02
bash build_episode.sh
```

This runs all 4 steps in sequence:
1. `tts.py` → `voice.wav` + `timing.json` (from Kokoro)
2. `build.py` → `animation.html` (with TOTAL_DURATION baked in)
3. `render.py` → `frames/` (exact frame count from timing.json)
4. `ffmpeg` → final `.mp4` (audio + video synced)

### 4. Verify Output

```bash
ffprobe -v error -show_format <series>/videos/ep<NN>.mp4 | grep duration
# Should match timing.json total_duration + small A/V sync buffer
```

### 5. Upload & Document

1. Upload `.mp4` to Discord in `#iclaw-e`
2. Get feedback
3. Once approved:
   - Commit production assets to git: `git add series/react/production/ep<NN>/ && git commit`
   - Update `series/react/episodes.md` (status = "approved")
   - Upload to YouTube with SRT captions
   - Add to playlist
   - Link to documentation page in video description

## Key Rules

- **No hardcoded durations.** Read `total_duration` from `timing.json`.
- **No guessed frame counts.** Read `total_frames_30fps` from `timing.json`.
- **Animation scenes reference TIMING array.** Use `TIMING[i].start` and `TIMING[i].end`.
- **Kokoro voice: bm_george** (British male, professional). Never change without approval.
- **Speed: 1.0** (normal). Adjust only if narration needs timing tweaks.
- **Intro silence: 2.0s** (before first sentence starts narration).
- **Outro buffer: 4.0s** (after last sentence, for final scene fade).
- **FPS: 30** (not 24, not 60). All timing calculations assume 30fps.

## Animation Quality Standards (MANDATORY)

Every episode MUST meet these visual standards. Study the gold standard references.

### Required visual elements per scene:
- **Icons/emoji** — each concept gets a visual anchor (🔘, 📤, ❌, ✅, etc.)
- **Semantic color coding** — consistent per concept type (green=correct, red=wrong, brand color=titles)
- **Caption overlay** — subtitle pill at bottom showing current narration text
- **Progressive reveal** — elements animate in with easing, not instant appear
- **Code blocks** — editor-style with window dots (red/yellow/green), syntax coloring
- **Scene routing with crossfades** — no jump cuts between scenes
- **Brand identity** — series logo (React atom, K8s helm, etc.) in hook + outro

### Scene structure pattern:
```
Scene 1: Hook (0–8s)     — logo reveal + title + subtitle
Scene 2-N: Content        — one concept per scene, cards/code/diagrams
Scene N+1: Recap          — bullet list of what you learned
Scene N+2: Outro          — Run the Docs end card
```

### Animation function pattern:
```javascript
// Per-scene function with time-based alpha
function drawScene2(ctx, t) {
  const s = TIMING[1].start;  // scene start from timing
  ctx.fillStyle = BG; ctx.fillRect(0,0,W,H); drawGrid(ctx);
  
  // Title with eased fade-in
  const titleA = ease.out(seg(t, s, s+0.4));
  // ... content with spring animations, progressive delays
}
```

1. Extract 1 frame per scene, visually inspect ALL of them
2. Check for black/empty gaps between scenes
3. Verify captions are rendering
4. Compare side-by-side with K8s reference frames

## Troubleshooting

**Video is longer/shorter than audio:**
- Check `render.py` is reading `total_frames_30fps` from `timing.json`
- Verify `timing.json` was generated by `tts.py` (not manually edited)
- Check ffmpeg mix command uses correct input audio

**Animation doesn't sync with narration:**
- Verify `animation.html` contains the `TIMING` array from `timing.json`
- Check `build.py` ran and embedded correct `TOTAL_DURATION`
- Verify animation uses `seg(t, TIMING[i].start, TIMING[i].end)` not hardcoded times

**Frames render but no animation:**
- Check browser console for JS errors (use Playwright error output)
- Verify `animation.html` has `window.renderFrame` function
- Check canvas element exists with id="c"

**Kokoro voice sounds wrong:**
- Verify `tts.py` uses `voice='bm_george'` and `lang='en-us'`
- Check `/Users/claude/runthedocs/kokoro/onnx/model.onnx` exists
- Check `/Users/claude/runthedocs/kokoro/voices.npz` exists
